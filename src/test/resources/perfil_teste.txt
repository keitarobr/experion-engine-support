Assim, a busca de informações nesse grande repositório de dados, a identificação de quais fontes de dados são confiáveis e a comparação de informações originadas a partir de fontes de dados diferentes tornaram-se tarefas do cotidiano, para as quais não se tem apoio suficiente. Contudo, a grande heterogeneidade na representação dos dados não permite que se identifique automaticamente quais valores se referem ao mesmo objeto, sendo necessária a participação do usuário no processo. Um dos problemas que surge em ambientes que fazem uso de funções de similaridade é como visualizar os resultados dessas funções e as alternativas que devem ser apresentadas aos usuários para decisão. O objetivo principal deste projeto é estudar métodos, técnicas e ferramentas focados nas necessidade dos usuários em visualizar e interagir com dados agregados a partir de funções de similaridade, dados incertos.

GeDiH Gerenciamento e disponibilização de dados Heterogêneos (Edital PQ - CNPq)	Os dados disponibilizados na Web são, reconhecidamente, o maior ambiente de dados heterogêneos da nossa atualidade, e este ambiente está em crescimento contínuo tanto em o volume quanto na complexidade de sua estrutura. Esta característica gera problemas nas mais diversas áreas e contextos, principalmente em integração e busca de informações. Devido à característica de larga escala (volume), um mesmo objeto do mundo real pode aparecer em vários contextos. Ao mesmo tempo, por tratar-se de dados altamente heterogêneos (complexidade), este mesmo objeto pode ter múltiplas representações em diferentes fontes de dados. Estas duas características, em particular, criam diversos problemas de manipulação dos dados na Web, tal como efetuar detecção das várias representações de um mesmo objeto da realidade. Apesar de muito ter sido estudado, e de diversas propostas terem sido apresentadas na comunidade científica, a alta heterogeneidade e o grande volume de dados existente são fatores críticos para o real funcionamento das propostas. O conceito de Dataspace Support Platform (DSSP) foi introduzido na comunidade como uma solução para gerenciar dados disponibilizados em ambientes com características de heterogeneidade e larga escala, como a Web. Basicamente, o objetivo de uma DSSP é conter componentes capazes de fornecer serviços inter-relacionados e que garantam acesso fácil a dados distribuídos. Partindo da ideia de DSSPs, a proposta desse projeto é a continuidade do projeto desenvolvido nos três últimos anos de Bolsa em Produtividade (de 2011 a 2013), através da especificação e desenvolvimento de técnicas e algoritmos de gerenciamento de dados com múltiplas representações, conectados entre si em um mesmo ambiente. Desta forma, será possível ter um ambiente provido de funcionalidades que irão permitir que usuários acessem os dados distribuídos em diversas fontes de dados heterogêneas. Mariage: Extração e casamento de dados estruturados da Web com múltiplas representações (Universal 2016)	A Web é o maior ambiente de dados heterogêneos da atualidade, onde o volume e a complexidade da estrutura destes dados estão em crescimento contínuo. Esta característica gera alguns problemas nas mais diversas áreas e contextos, principalmente em integração e busca de informações. Devido à característica de larga escala (volume), um mesmo objeto do mundo real pode aparecer em vários contextos. Ao mesmo tempo, por tratar-se de dados altamente heterogêneos (complexidade), este mesmo objeto pode ter múltiplas representações em diferentes fontes de dados. Estas duas características, em particular, criam Página 1 de 5 diversos problemas de manipulação dos dados na Web, tais como consultas, integração, indexação entre outros. Dentro deste contexto, a Web é considerada também o maior repositório de dados estruturados existentes, considerando apenas Web Tables e Web Lists. A Web possui mais de 150 milhões de tabelas HTML com alta qualidade de conteúdo considerado ?dado relacional?. A principal motivação para trabalhar com este tipo de informação é possibilitar análise e integração de dados na Web, pois ela pode ser considerada a maior base de conhecimento existente. Muitos trabalhos têm sido desenvolvidos para tratar Web Tables, e dentre eles podem-se destacar identificação de tabelas e extração de dados, a fim de que sejam utilizado como base para consultas estruturadas na Web. O grande problema é que a grande maioria das tabelas Web possui estrutura diferente das tabelas relacionais, e portanto precisam ser individualmente analisadas a fim de sua estrutura possa ser compreendida. Além da heterogeneidade estrutural, podem ser extraídas tabelas com o mesmo conteúdo de diferentes fontes de dados, voltando ao problema de múltiplas representações do mesmo objeto do mundo real. Como não é possível assegurar com exatidão que duas representações se referem ao mesmo objeto, a solução é prover um mecanismo que forneça uma noção de proximidade entre os valores, usando funções de similaridade. De forma geral, o termo usado para designar esta questão é casamento aproximado. Assim sendo, dentro deste contexto, três tarefas são muito importantes para a proposta de uma solução: (i) análise das estruturas dos dados a fim de identificar conteúdo útil; e (ii) casamento (matching) de dados extraídos de diversas fontes de dados; e (iii) criação de um esquema que represente a estrutura dos dados extraídos. Na tarefa de análise das estruturas, se faz necessário o desenvolvimento de um crawler que identifique dados estruturados na Web, de forma que seja possível extrair o conteúdo de cada um deles. O desafio é definir uma abordagem precisa e eficaz para a determinação de equivalências entre as muitas possíveis estruturações que um mesmo dado pode ter na Web. A intenção é definir uma técnica mais precisa do que as atualmente definidas pela comunidade. A tarefa de definição de esquema deverá gerar como resultado um esquema que apresente uma visão da estrutura extraída das diversas fontes de dados. Outro ponto importante, que deve ser considerado no algoritmo de matching, é a evolução temporal dos dados, dada pela atualização constante dos dados.
Aglomerante: Ambiente para Gerenciamento de Dados com Múltiplas Representações (Edital PQ - CNPq)	Muito tem sido estudado e diversas propostas têm sido apresentadas dentro da comunidade de Banco de Dados, no sentido de adaptar as técnicas já maduras de Sistemas Gerenciadores de Banco de Dados para o ambiente da Web. No entanto, dada sua alta heterogeneidade e o grande volume de dados existente, técnicas tradicionais de banco de dados não podem ser aplicadas diretamente. Recentemente, para tentar resolver esse problema, o conceito de Dataspace foi introduzido na comunidade. Dataspace pode ser definido como uma abstração do conceito de gerenciamento de dados, em que as tarefas de manipulação dos dados são bastante flexíveis em relação à sua natureza e estrutura. Juntamente com este conceito, foi introduzida uma arquitetura denominada Dataspace Support Platform (DSSP), cujo objetivo é habilitar componentes capazes de fornecer serviços inter-relacionados e que garantam acesso fácil a dados distribuídos em um Dataspace. Partindo da idéia de DSSPs, a proposta desse projeto é a especificação e desenvolvimento de técnicas e algoritmos de gerenciamento de dados com múltiplas representações, conectados entre si em um ambiente chamado Aglomerante. O projeto contempla como principal contribuição a especificação e construção de técnicas e algoritmos de gerenciamento de dados com múltiplas representações.
